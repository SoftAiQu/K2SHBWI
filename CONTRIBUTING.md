# Contributing to K2SHBWI

Thank you for your interest in contributing to K2SHBWI! We welcome contributions from everyone.

## Quick Start

1. **Read the guidelines:** See [`docs/05-contributing/`](docs/05-contributing/)
2. **Set up environment:** Follow [`docs/05-contributing/02-development-setup.md`](docs/05-contributing/02-development-setup.md)
3. **Make changes:** Follow [`docs/05-contributing/04-code-standards.md`](docs/05-contributing/04-code-standards.md)
4. **Submit PR:** Follow [`docs/05-contributing/05-pull-request-process.md`](docs/05-contributing/05-pull-request-process.md)

## How to Contribute

### Code
- Fix bugs
- Add features
- Improve performance
- Write tests

### Documentation
- Write guides
- Improve READMEs
- Create examples
- Translate docs

### Community
- Answer questions
- Report bugs
- Suggest improvements
- Help others

## Development Setup

```bash
# Clone repository
git clone https://github.com/k2shbwi/k2shbwi.git
cd k2shbwi

# Create virtual environment
python -m venv venv
source venv/bin/activate  # Windows: venv\Scripts\activate

# Install dependencies
pip install -r requirements.txt

# Run tests
pytest tests/
```

## Code Standards

- Follow PEP 8
- Use type hints
- Write docstrings
- Add tests for new features

## Reporting Bugs

1. Check existing issues first
2. Create new issue with:
   - Clear title
   - Step-by-step reproduction
   - Expected vs actual behavior
   - Python version & OS

## Suggesting Features

1. Check roadmap: [`docs/04-roadmap/`](docs/04-roadmap/)
2. Create issue with:
   - Clear description
   - Use cases
   - Examples

## Community Guidelines

- Be respectful & inclusive
- No harassment or discrimination
- Respect others' time & effort
- Assume good intent

See [`CODE_OF_CONDUCT.md`](CODE_OF_CONDUCT.md) for details.
