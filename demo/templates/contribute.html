Contribute to K2SHBWI

Help us build the future of image compression. All contributions are welcome!

Getting Started
1. Fork & Clone

Start by forking the repository on GitHub and cloning it locally:

git clone https://github.com/your-user/k2shbwi.git
2. Setup Environment

Create a Python virtual environment and install dependencies:

python -m venv venv && source venv/bin/activate
3. Install Dependencies

Install all required packages:

pip install -r requirements.txt
4. Create a Branch

Create a feature branch for your changes:

git checkout -b feature/your-feature

How K2SHBWI Compression Works

1
Analysis

Input image is analyzed for color patterns and frequency components

2
Transformation

DCT/Wavelet transformation decomposes image into components

3
Quantization

Components are quantized based on perceptual importance

4
Encoding

Huffman/Arithmetic coding produces compressed output

Development Tips & Best Practices

Performance Matters

Use profiling tools. Every millisecond counts in compression algorithms. Benchmark before/after changes.

Test Thoroughly

Write unit tests for new code. Test with diverse image types. Maintain >90% code coverage.

Document Well

Add docstrings to functions. Keep README updated. Explain complex algorithms with comments.

Follow Code Style

Use Black for formatting. Follow PEP 8 guidelines. Run flake8/pylint before committing.

Commit Wisely

Use atomic commits with clear messages. Reference issues in commits. Keep commits focused.

Review Process

Create PRs for all changes. Be open to feedback. Help review others' PRs. Be respectful.

Pull Request Process

  1. Create descriptive PR title: "Fix: Memory leak in encoder" or "Feature: GPU acceleration for DCT"
  2. Write PR description: Explain what changes, why they matter, and how to test them
  3. Pass all tests: All GitHub Actions CI/CD checks must pass
  4. Add tests & documentation: New features must include tests and updated docs
  5. Wait for review: At least one maintainer must review and approve
  6. Squash & merge: Maintainers will merge and add entry to changelog
